Fabric

a data layer for the distributed web

Introduction

Building products that fully utilize the power of emerging technology is hard. That's why we built Fabric, a standardized API for designing and deploying decentralized apps.

Building Your First Decentralized Application

We'll be using Maki, a library of user interface elements which integrate directly with Fabric. We'll provide definitions for how our application should behave, and the core Fabric library will handle the rest.

If you haven't already, now's a good time to read an Introduction to Maki which details available options and configurations which might be interesting to you.

Basic App Scaffold

Start with creating the following app.js file:

          var Fabric = require('fabric');
          var Maki = require('maki');
          
          var app = new Maki();
          
          app.start();

This defines and starts a simple HTTP Oracle, and displays a link in your console. You can browse this link now, or read on to add some functionality.

Adding a Data Layer

Fabric adds a simple synchronized datastore to your applications, allowing your users to keep a collection of the data they've consumed.

This data can be used to run applications locally, without the need for a network connection. Let's add one now.

Place this code before app.start(); to add a new Post resource:

          app.define('Post', {
            attributes: {
              content: { type: String }
            }
          });
